Introduction
Project Overview

This behavior runs on a NAO/Pepper robot using Choregraphe (SoftBank Robotics' visual programming IDE) and NAOqi (the robot's operating system and API framework).

The behavior does two things: Learns a face and stores it in the robot's internal database, then Recognizes that face later and greets the person by name.

4

5
Say Boxes
Known face → "Hello antonio!"  /  Unknown → "I don't know you yet"
02
Core API
ALFaceDetection — The Core Module

Everything depends on this NAOqi module. You access it via a proxy:

Python · ALProxy
self.faceDetectionModule = ALProxy("ALFaceDetection")
What ALProxy does: It creates a local Python object that communicates over the network with the actual module running on the robot. The string "ALFaceDetection" is the registered service name in NAOqi's service directory.

Key methods on this proxy:

Key Methods
# Store current face in database linked to "name"
# Returns True on success, False on failure
self.faceDetectionModule.learnFace("antonio")

# Track detected faces with the robot's head motors
self.faceDetectionModule.setTrackingEnabled(True)

# 80% confidence required before confirming recognition
self.faceProxy.setRecognitionConfidenceThreshold(0.8)
03
Event System
ALMemory — The Event Bus

NAOqi uses a publish-subscribe event model through ALMemory. Think of it as a global shared blackboard:

Camera detects face → writes data to ALMemory["FaceDetected"] → fires event → FaceReco. box receives it

The FaceDetected data structure is a nested array:

ALMemory FaceDetected Structure
p = [
  timestamp,          # [0] when the event fired
  [                   # [1] list of all detected faces
    [
      faceInfo,       # raw detection parameters
      [               # extra info array
        label,        # [0] name string e.g. "antonio"
        confidence,   # [1] float 0.0 → 1.0
        ...
      ]
    ],
  ],
  ...
]

# Get most recent face — take last element of p[1]
result = p[1][len(p[1])-1]
4
Detected — Not Recognized
Face seen in camera but has no match in database. Fires "stranger" path.
2 / 3
Recognized ✓
Face matched to stored entry above confidence threshold. Fires greeting path.

Structure
Choregraphe Box Lifecycle

Every box in Choregraphe runs a Python class inheriting from GeneratedClass. There are four key methods:

Box Lifecycle Template
class MyClass(GeneratedClass):
    def __init__(self):
        GeneratedClass.__init__(self, False)
        # False = don't log every signal — runs once on load
        self.faceDetectionModule = ALProxy("ALFaceDetection")

    def onLoad(self):
        # Called when behavior STARTS — initialize state here
        self.bIsRunning = False

    def onUnload(self):
        # Called when behavior STOPS — clean up resources
        self.bIsRunning = False

    def onInput_onStart(self):
        # Called when "onStart" input wire fires — main entry point
        pass

# Output methods — how a box sends signals forward:
self.onStopped()              # fire output with no data
self.onStopped("antonio")    # fire output WITH data
self.onSuccess()              # fire success output
self.onFailure()              # fire failure output

Deep Dive
Stage-by-Stage Code Walkthrough

STAGE 2a — Only Once Box

Ensures the learning pipeline triggers exactly once per behavior run using a boolean flag. Without this, rapid event firing could call learnFace() multiple times simultaneously.

Only Once
def onLoad(self):
    self.bFirstTime = True

def onInput_onSignal(self, p):
    if self.bFirstTime:         # first call: allow through
        self.bFirstTime = False  # flip — never True again
        self.firstTime(p)          # fire output with data p
    # subsequent calls: if block skipped, nothing happens

STAGE 2b — Wait Box (5 seconds)

qi.async schedules a function call after a delay without blocking the main thread. Delay is in microseconds — multiply seconds × 1,000,000.

Wait Box
def onInput_onStart(self):
    self.waiting = qi.async(
        self.triggerOutput,
        delay=int(self.getParameter("Timeout (s)") * 1000 * 1000)
    )   # runs triggerOutput() after 5 seconds on background thread

def onUnload(self):
    if self.waiting:
        self.waiting.cancel()  # cancel if behavior stops early

STAGE 2c — Eyes LEDs Box

post.fadeRGB is asynchronous (returns an ID immediately). leds.wait(id, 0) then explicitly blocks until the animation finishes. Color is packed as a 24-bit integer: R×65536 + G×256 + B.

Eyes LEDs · Color Encoding
def onInput_color(self, p):
    # p = [red, green, blue] each 0.0 → 1.0
    sGroup = "FaceLeds"  # or LeftFaceLeds / RightFaceLeds

    # Pack RGB floats → single 24-bit integer
    colorInt = int(256*256*p[0] + 256*p[1] + p[2])

    # post.fadeRGB = async call, returns ID immediately
    id = self.leds.post.fadeRGB(sGroup, colorInt, duration)
    self.ids.append(id)
    self.leds.wait(id, 0)  # NOW block until animation done
    self.ids.remove(id)

    if self.ids == []:
        self.onDone()

STAGE 2d — WaitToSend Box (Synchronization Gate)

Implements a two-condition gate (join node). Only fires output when both conditions are met regardless of order: (1) name string arrived + (2) 5-second timer fired.

WaitToSend · Synchronization Gate
def onLoad(self):
    self.ok1 = None   # stores the name string
    self.ok2 = False  # True if timer already fired

def onInput_onMessage(self, p):   # name "antonio" arrives
    if not self.ok2:
        self.ok1 = p        # timer not yet fired → store name
    else:
        self.ok(p)          # timer already fired → GO!

def onInput_onSignal(self):         # 5-second timer fires
    if self.ok1 is None:
        self.ok2 = True     # name not yet here → set flag, wait
    else:
        self.ok(self.ok1)    # name already stored → GO!

STAGE 2e — Inner Learn Face Box ⭐ THE CORE

Calls ALFaceDetection.learnFace() which captures the current camera frame, extracts facial features, and stores them in the robot's database linked to the name string. Uses try/except because hardware APIs can fail at any time.

learnFace() — Core Learning Logic
def onInput_onLearn(self, p):
    # p = "antonio"

    # Guard clause — prevent concurrent calls (lightweight mutex)
    if self.bIsRunning:
        return
    self.bIsRunning = True

    self.logger.info("LearnFace called with name: " + str(p))

    try:
        # Enable head tracking so robot follows the face
        self.faceDetectionModule.setTrackingEnabled(True)

        # THE CORE CALL:
        # Snapshot → extract features → store in database as "antonio"
        result = self.faceDetectionModule.learnFace(p)

        self.logger.info("learnFace result: " + str(result))

        if result:
            self.onSuccess()   # Eyes turn GREEN
        else:
            self.onFailure()   # Eyes turn RED

    except Exception as e:
        # NAOqi disconnection, module crash, no face in frame…
        self.logger.error("learnFace exception: " + str(e))
        self.onFailure()

    self.bIsRunning = False  # reset guard

STAGE 3 — Face Reco. Inner Box

Fires every time ALMemory's FaceDetected event triggers. Parses the nested face data array, checks the status code, and routes to the correct output.

Recognition Logic
def onInput_onStart(self, p):
    # Set 80% confidence threshold
    self.faceProxy.setRecognitionConfidenceThreshold(0.8)

    if len(p) > 0 and len(p[1]) > 0:
        # Get most recent frame's face data
        result = p[1][len(p[1])-1]

        if len(result) == 1:
            if result[0] == 4:              # SEEN, not recognized
                self.onDetectWithoutReco()

        elif len(result) == 2:
            if result[0] in [2, 3]:         # RECOGNIZED above 80%
                for name in result[1]:
                    self.onRecognizedFace(name)  # sends "antonio"

STAGE 4 — Say Boxes

ALTextToSpeech.say() is synchronous — blocks until robot finishes speaking. Use post.say() for non-blocking speech. str(p) converts p safely even if it is not already a string.

Say Boxes
# Known face → receives "antonio" as parameter p
def onInput_onStart(self, p):
    sentence = "Hello " + str(p) + ", I recognize you!"
    self.tts.say(sentence)   # SYNCHRONOUS — waits until done
    self.onStopped()

# Unknown face → no parameter needed
def onInput_onStart(self):
    sentence = "I see you, but I do not know your name yet."
    self.tts.say(sentence)
    self.onStopped()

Theory
Key Technical Concepts

These are the design patterns and CS concepts your professor will want to hear you reference:

Proxy Pattern
ALProxy is the Proxy design pattern — a local object that transparently forwards calls to a remote service. The caller doesn't care if it's local or over a network.
Pub-Sub Architecture
FaceDetected ALMemory event is publish-subscribe. The camera pipeline publishes events; any number of subscribers can react independently.
Barrier Synchronization
WaitToSend is a join node — don't proceed until multiple independent conditions are all satisfied. Same concept as a barrier in concurrent programming.
Guard Clause + Mutex
bIsRunning flag is a lightweight mutual exclusion mechanism preventing re-entrant calls to learnFace(), which is not thread-safe.
Precision vs Recall
Setting 0.8 threshold is a classic ML trade-off. Higher = fewer false positives, more false negatives. Lower = more matches, more wrong recognitions.
Defensive Programming
Try/except for hardware APIs. In robotics, hardware can fail at any time (WiFi, module crash, face lost). Graceful failure is critical — a crash freezes the robot mid-interaction.
